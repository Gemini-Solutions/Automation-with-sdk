Key steps -------------------------------------------------------------------------------------------------------- 
1. UGTF/ python code/ engine parses the xml, convert in to desired dict format, and passes it to the runner.
2. python code will send create object of engine, pass the config and tc name to engine.
3. (for cli run) Engine then takes out the data for that testcase from config, formats config, converts/ filters it to required dict
4. Engine then creates object of RESTTEST runner

Runner ------------------------------------------------------------------------------------------------------
1. creates an object of pygem report
2. get necessary data from config data
3. validates data
4. Forms request using api_common
5. gets response, validates response
6. Adds rows to report
7. formats the result
8. Will display the result file
9. Return to UGTF the required output
